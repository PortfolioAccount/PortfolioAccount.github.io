자바를 선택하게 된 이유

처음에는 C로 코딩을 시작하였지만 재미로 만들던 콘솔로 간단한 게임을 만들던 도중 C로 구현을 하는 것에 한계를 느꼈고, 다른 언어를 찾아보던 도중 자바의 정석이라는 책을 통해 자바를 접하게 되었고 자바에 매력에 빠지게 되었습니다. 다른 사람들이 만든 다양한 소스들과 JVM의 매력, 다양한 곳에서 사용되고 다양한 곳에 응용되는 것에 자바는 저에게 최고의 언어였습니다.

활동


    SQLite

    Rails, Android 에서 사용하였습니다. Rails는 원래 MySQL를 사용할 생각이였지만 기존에 설치되었던 MariaDB와 충돌이 일어나 SQLite를 선택하게 되었습니다.


    MySQL

    데이터 베이스 중 MariaDB를 배우면서 공부하게 되었습니다.


    MariaDB

    무료로 사용할 수 있다는 장점과 DBMS등의 오픈 소스가 존재했기에 사용하였던 디비 입니다. 큰 프로젝트는에는 적용하지 않았지만 터미널을 이용하여 접근등의 노력을 꾸준히 하고 있습니다.


    AndroidStudio

    취미로 시작하게 된 기술 입니다. 앱을 만들어 실제로 스토어에 등록까지 하면서 실력이 어느정도 쌓이게 되었습니다. (앱 다운로드 링크)


    Arduino

    학교 졸업 작품에서도 이용하였으며 이전에도 관련된 학습을 해왔습니다. (VR Project 링크)


    Unity

    학교 졸업 작품을 위해 VR 맵 제작을 위해 학습을 하였습니다. (VR Project 링크)


    Gradle

    Maven보다는 주로 Gradle를 사용하며 Android Studio, Spring 등의 다양한 곳에서 사용되기 때문에 사용하게 되었습니다.


    OpenCV

    OCR 기능 및 색깔 추출, 얼굴 인식등의 프로그램을 구현하기 위해 이용하였습니다.


    Firebase

    안드로이드 스튜디오를 공부할때 이용하기 위해서 연습하였습니다. 구현에는 성공하였지만 많은 데이터양으로 인해서 요금제 부담으로 인해서 실제로 적용하진 못했습니다.


    Apache Tomcat

    Spring을 공부하면서 알게 되었고 학습하게 되었습니다.


    Apache Tomcat

    Spring을 공부하면서 알게 되었고 학습하게 되었습니다.


    HTML5

    기존에 Spring과 이 포토폴리오를 구현하면서 자세하게 알게 되었고 학습하게 되었습니다. 하지만 포토폴리오에 내가 원하는데로 구현한것이 나오지 못해 적용하지 못한 등의 부족함이 느껴졌습니다.


    CSS3

    해당 포토폴리오를 구현하기 위해 간단한 학습을 하였습니다.


    Git

    github를 공부하면서 학습하게 되었습니다.


    GitHub

    처음에는 백업을 목적으로 사용하였지만 이후 오픈 프로젝트에 관심이 생겨서 활동하게 되었습니다. 그 밖에도 잔디 심기나 프로젝트 공유등의 재미를 느끼게되었습니다.


    Notion

    처음에는 주변사람의 추천으로 알게 되었지만 마크다운에 빠져있던 저에게는 타이밍이 좋지 못한것 같았습니다. Notion의 활동기간은 짧았고 이후 Velog를 통해서 활동을 시작하게 되었습니다.


    Velog

    저의 활동을 메모하고 공유하고 싶어서 시작하게 되었습니다. 깃허브에서 마크다운을 알게되고 이에 익숙한 Velog는 저에게 친숙하게 느껴졌습니다. 이후 글을 쓰게 되었는데 글의 퀄리티가 아쉽다는 것이 느껴졌습니다.
